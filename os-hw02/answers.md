#  Вопрос 1
Что произойдёт, если в скрипте без set -e команда ls /nonexistent вернёт ошибку? Продолжит ли скрипт выполнение? Почему это опасно?
Скрипт продолжит выполнение без set -e даже если в ней будут какие то ошибки, что может привести 
к нежелательным последствиям и даже потере данных. Поэтому всегда важно писать set -e чтобы в случае чего
всегда можно было исправить ошибку
# Вопрос 2 
Чем отличается [[ -f "$1" ]] от [ -f "$1" ]? В каком случае второй вариант может сломаться?
Двойные скобки отличаются от одинарных тем, что первые
безопасны с пробелами, поддерживают &&, ||, regex. [ ] будет ломаться если $var пуст
# Вопрос 3 
Почему for file in $(ls *.log) — anti-pattern? Какой правильный способ перебрать файлы?
потому что при таком подходе имена файлов, содержащие пробелы могут неожиданно выдать ошибку, а это
нежелательно. В таком случае правильно использовать glob
for file in *.log; do
    echo "$file: $(wc -l < "$file") строк"
done
