# Гипотеза: Чего я ожидаю?
я ожидаю в первую очередь что вывод для nice -10 будет быстрее чем для nice 19, 
потому что в таком случае на него тратиться больше ресурсов CPU. По приоритету выше те процессы, чей nice меньше

# Результаты
nice -10
real    0m 0.052s
user    0m 0.001s
sys     0m 0.013s
nice 19
real    0m 3.390s
user    0m 0.005s
sys     0m 0.008s

Как мы видим в этом примере процесс с приоритетом nice -10 реально выполнил задачу быстрее, кроме sys
sys в этом случае мы можем не учитывать, потому что в этом случается идет с помощью процессора 
запись и чтение с диска, что от nice никак не зависит

# Почему так вообщен происходит? Что делает CFS? 
CFS — Completely Fair Scheduler, следит за тем, чтобы каждый из процессов получал справедливую долю 
CPU. У каждого процесса есть свой vruntime, это такой счетчик по которому CFS определяет какому процессу
нужнее всего CPU - а именно тому, у которого этот счетчик меньше. nice влияет на скорость этого счетчика -
для nice -10 он увеличивается медленнее, в то время как для nice 10 он наоборот накапливается быстрее из за 
чего процесс остается дольше в очереди

# Что изменится без фоновой нагрузки?
Без фоновой загрузки процессы будут просто распределены между ядрами, вследствии чего конкуренции 
за CPU не будет и они просто выполнят свою задачу не полагаясь на vruntime. Это происходит потому что 
каждому из процессов нужны ресурсы, а если их не хватает то им приходится стоять в очереди, с чем помогает
CFS
