# Вопрос 1
Что такое `strace`? Для чего инженер может использовать `strace` на продакшн-сервере?
strace — это утилита, которая перехватывает и показывает все системные вызовы (syscalls), которые программа делает к ядру операционной системы.
Через системные вызовы программы могут с помощью ядра Линукс обращаться к жесткому диску для выполнения каких то привелегированных операций: чтение,
вывод на экран.
На практике инженеры используют strace для нескольких вещей

# Поиска ошибок. 
Например, если программа падает с ошибкой доступа, strace покажет, к какому именно файлу она пыталась обратиться и с какими правами.
# Анализа производительности.
С ключом -c можно собрать статистику (сколько времени занял каждый вызов и где программа "тормозит").
# Понимания работы: 
Чтобы увидеть, что на самом деле делает утилита "под капотом", а не только то, что написано в документации.

#  Вопрос 2
Что показывает `lsof -p PID`? Чем это отличается от `ls /proc/PID/fd/`?
lsof -p PID показывает все открытые файлы процесса с расширенной информацией: тип файла, доступ, размер, inode
ls /proc/PID/fd/ показывает только файловые дескрипторы

# Вопрос 3
Что такое `/proc`? Это реальная файловая система на диске?
/proc это виртуальная файловая система, котораяне хранится на жестком диске. Это интерфейс ядра Linux для реального времени 
информации о процессах, памяти, CPU.
# Вопрос 4
Что делает `tee`? Напишите пример команды, которая выводит `ls /etc` на экран и одновременно записывает в файл `output.txt`.
tee читает из stdin, пишет в stdout и одновременно в файлы. Это раздвоение потока, с выводом на экран и одновременной 
записью в файл. Вот примеры как можно использовать tee с выводом ls /etc на экран и записью в output.txt:

# Дописать в существующий файл
ls /etc | tee -a output.txt

# Несколько файлов  
ls /etc | tee file1.txt file2.txt > /dev/null

# С правами root
ls /etc | sudo tee output.txt
