# 1) Насчет pitfall
Разница с set -euo pipefail довольно очевидна: в первом случае когда мы не поставили эту команду,
даже после вывода ошибки с "не найденной директорией" код продолжил выполняться. Во втором же случае
set -euo pipefail просто не позволила коду исполняться далее. 
Я думаю это особо полезно когда в коде сотня-тысяча
строчек которые завязаны на директории и если что то пойдет не так то будет выводиться не один десяток ошибок
в консоль, что не очень хорошо, особенно когда отдельные блоки кода связаны между собой. В этом же случае ошибка
покажется только раз и ее можно будет исправить

# Тесты с fileinfo:
1)Без аргументов ./fileinfo   -   Использование: ./fileinfo.sh <файл>

2)Несуществующий файл ./fileinfo.sh /tmp/no_such_file    -   Ошибка: '/tmp/no_such_file' не найден или не является файлом

3)Неисполняемый файл ./fileinfo.sh /etc/passwd       
=== Информация о файле ===
Имя: /etc/passwd
Строк: 28
Размер: 4.0K
Исполняемый: нет

4)Исполняемый файл ./fileinfo.sh greet.sh
=== Информация о файле ===
Имя: greet.sh
Строк: 7
Размер: 4.0K
Исполняемый: да
