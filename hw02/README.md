# Как настроить cron на автоматические срабатывание backup.sh каждые 3 часа и не облажаться

# 1) Описание backup.sh
Для начала расскажу про то, как пользоваться backup.sh. В структуре проекта мы уже разобрали
что backup.sh - это скрипт, который сохраняет архив любой директории в backup.sh. Для его запуска, в 
bash нам нужно написать:
./backup.sh <имя нужной директории>

После чего backup.sh выведет отчет о сохраненном архиве. В отчете указан источник директории,
название архива(вместе с датой), размер в КБ, название директории в которой будет лежать архив и 
количество имеющихся архивов в backups

$ ./backup.sh ~/os-systems-work/hw-01/

=== РЕЗЕРВНОЕ КОПИРОВАНИЕ ===
Источник: /home/melted/os-systems-work
Архив: backup-2026-04-26_22-11-11.tar.gz
Размер: 2392
Сохранено в /home/melted/os-systems-work/os-hw02/backups
Архивов после ротации: 3
=== Готово! ===


# 2)Требования:
 перед работой с cron вам нужно установить systemd в WSL2
если у вас его нет. Вот подробная инструкция как можно это сделать

Проверка:
ps -p 1 -o comm=        Если в выводе будет systemd - Все ок

Если же у нас не установлен systemd
sudo nano /etc/wsl.conf

В открывшемся поле пишем:
[boot]
systemd=true

Сохраните (Ctrl+O, Enter, Ctrl+X), затем в PowerShell
wsl --shutdown

и снова проверьте

# 3)Работа с crontab
Открываем:      crontab -e
В окне GNU nano пишем: 0 *3 * * * /home/<ваше имя>/hw02/backup.sh /home/username/documents - запись каждые 3 часа

Чтобы закрыть и сохранить окно - Ctrl+O, Enter, Ctrl+X

# 4)Как удостовериться что cron работает?
есть несколько способов проверить что cron работает.

I)Проверка статуса службы. Можно у системы спросить ее состояние. Если в Shell набрать
systemcl status cron и он покажет зеленый то все отлично работает

II)Поиск процесса в памяти. Среди всех запущенных процессов можно поискать cron в Shell
ps aux | grep cron

III)Просмотр системного журналаю Cron сам пишет отчеты о запуске любой задачи
grep CRON /var/log/syslog

# Как отключить cron
crontab -r удаляет ВСЕ задачи
crontab -e чтобы проверить есть ли у вас задачи
crontab -l покажет активные задачи для crontab

также можно зайти в crontab crontab -e и просто удалить строчку с вашей задачей
